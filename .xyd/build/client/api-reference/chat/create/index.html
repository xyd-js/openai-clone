Create chat completion

POST
https:/api.openai.com/v1/chat/completions

Starting a new project? We recommend trying Responses to take advantage of the latest OpenAI platform features. Compare Chat Completions with Responses.

Creates a model response for the given chat conversation. Learn more in the text generation, vision, and audio guides.

Parameter support can differ depending on the model used to generate the response, particularly for newer reasoning models. Parameters that are only supported for reasoning models are noted below. For the current state of unsupported parameters in reasoning models, refer to the reasoning guide.

Request body

  • messages
    array of
    Required

    A list of messages comprising the conversation so far. Depending on the model you use, different message types (modalities) are supported, like text, images, and audio.

    • Developer message
      object
      Required
      Developer-provided instructions that the model should follow, regardless of messages sent by the user. With o1 models and newer, `developer` messages replace the previous `system` messages.
      • content
        string or array
        Required
        The contents of the developer message.
        • Text content
          string
          Required
          The contents of the developer message.
        • Array of content parts
          array
          Required
          An array of content parts with a defined type. For developer messages, only type `text` is supported.
          • object
            Required
            Learn about [text inputs](/docs/guides/text-generation).
            • type
              enum
              Required
              The type of the content part.
              • text
                string
            • text
              string
              Required
              The text content.
      • role
        enum
        Required
        The role of the messages author, in this case `developer`.
        • developer
          string
      • name
        string
        An optional name for the participant. Provides the model information to differentiate between participants of the same role.
    • System message
      object
      Required
      Developer-provided instructions that the model should follow, regardless of messages sent by the user. With o1 models and newer, use `developer` messages for this purpose instead.
      • content
        string or array of
        Required
        The contents of the system message.
        • Text content
          string
          Required
          The contents of the system message.
        • Array of content parts
          array of
          Required
          An array of content parts with a defined type. For system messages, only type `text` is supported.
          • Text content part
            array
            Required
            An array of content parts with a defined type. For developer messages, only type `text` is supported.
            • object
              Required
              Learn about [text inputs](/docs/guides/text-generation).
              • type
                enum
                Required
                The type of the content part.
                • text
                  string
              • text
                string
                Required
                The text content.
      • role
        enum
        Required
        The role of the messages author, in this case `system`.
        • system
          string
      • name
        string
        An optional name for the participant. Provides the model information to differentiate between participants of the same role.
    • User message
      object
      Required
      Messages sent by an end user, containing prompts or additional context information.
      • content
        string or array of
        Required
        The contents of the user message.
        • Text content
          string
          Required
          The text contents of the message.
        • Array of content parts
          array of
          Required
          An array of content parts with a defined type. Supported options differ based on the [model](/docs/models) being used to generate the response. Can contain text, image, or audio inputs.
          • Text content part
            array
            Required
            An array of content parts with a defined type. For developer messages, only type `text` is supported.
            • object
              Required
              Learn about [text inputs](/docs/guides/text-generation).
              • type
                enum
                Required
                The type of the content part.
                • text
                  string
              • text
                string
                Required
                The text content.
          • Image content part
            object
            Required
            Learn about [image inputs](/docs/guides/vision).
            • type
              enum
              Required
              The type of the content part.
              • image_url
                string
            • image_url
              object
              Required
              • url
                string
                Required
                Either a URL of the image or the base64 encoded image data.
              • detail
                enum
                Defaults: auto
                Specifies the detail level of the image. Learn more in the [Vision guide](/docs/guides/vision#low-or-high-fidelity-image-understanding).
                • auto
                  string
                • low
                  string
                • high
                  string
          • Audio content part
            object
            Required
            Learn about [audio inputs](/docs/guides/audio).
            • type
              enum
              Required
              The type of the content part. Always `input_audio`.
              • input_audio
                string
            • input_audio
              object
              Required
              • data
                string
                Required
                Base64 encoded audio data.
              • format
                enum
                Required
                The format of the encoded audio data. Currently supports "wav" and "mp3".
                • wav
                  string
                • mp3
                  string
          • File content part
            object
            Required
            Learn about [file inputs](/docs/guides/text) for text generation.
            • type
              enum
              Required
              The type of the content part. Always `file`.
              • file
                string
            • file
              object
              Required
              • filename
                string
                The name of the file, used when passing the file to the model as a string.
              • file_data
                string
                The base64 encoded file data, used when passing the file to the model as a string.
              • file_id
                string
                The ID of an uploaded file to use as input.
      • role
        enum
        Required
        The role of the messages author, in this case `user`.
        • user
          string
      • name
        string
        An optional name for the participant. Provides the model information to differentiate between participants of the same role.
    • Assistant message
      object
      Required
      Messages sent by the model in response to user messages.
      • content
        string or array of or null
        The contents of the assistant message. Required unless `tool_calls` or `function_call` is specified.
        • Text content
          string
          The contents of the assistant message.
        • Array of content parts
          array of
          An array of content parts with a defined type. Can be one or more of type `text`, or exactly one of type `refusal`.
          • Text content part
            array
            Required
            An array of content parts with a defined type. For developer messages, only type `text` is supported.
            • object
              Required
              Learn about [text inputs](/docs/guides/text-generation).
              • type
                enum
                Required
                The type of the content part.
                • text
                  string
              • text
                string
                Required
                The text content.
          • Refusal content part
            object
            • type
              enum
              Required
              The type of the content part.
              • refusal
                string
            • refusal
              string
              Required
              The refusal message generated by the model.
      • refusal

  • logit_bias
    object or null
    Defaults: null
    Modify the likelihood of specified tokens appearing in the completion. Accepts a JSON object that maps tokens (specified by their token ID in the tokenizer) to an associated bias value from -100 to 100. Mathematically, the bias is added to the logits generated by the model prior to sampling. The exact effect will vary per model, but values between -1 and 1 should decrease or increase likelihood of selection; values like -100 or 100 should result in a ban or exclusive selection of the relevant token.
